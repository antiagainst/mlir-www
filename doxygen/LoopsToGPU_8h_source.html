50 /// Convert a loop operation into a GPU launch with the values provided in
51 /// `numWorkGroups` as the grid size and the values provided in `workGroupSizes`
52 /// as the block size. Size of `numWorkGroups` and workGroupSizes` must be less
53 /// than or equal to 3. The loop operation can be an imperfectly nested
54 /// computation with the following restrictions:
55 /// 1) The loop nest must contain as many perfectly nested loops as the number
56 /// of values passed in through `numWorkGroups`. This corresponds to the number
57 /// of grid dimensions of the launch. All loops within the loop nest must be
58 /// parallel.
59 /// 2) The body of the innermost loop of the above perfectly nested loops, must
60 /// contain statements that satisfy one of the two conditions below:
61 /// a) A perfect loop nest of depth greater than or equal to the number of
62 /// values passed in through `workGroupSizes`, i.e. the number of thread
63 /// dimensions of the launch. Loops at depth less than or equal to size of
64 /// `workGroupSizes` must be parallel. Loops nested deeper can be sequential
65 /// and are retained as such in the generated GPU launch code.
66 /// b) Statements that are safe to be executed by all threads within the
67 /// workgroup. No checks are performed that this is indeed the case.
68 /// TODO(ravishankarm) : Add checks that verify 2(b) above.
69 /// The above conditions are assumed to be satisfied by the computation rooted
70 /// at `forOp`.
71 LogicalResult convertLoopToGPULaunch(loop::ForOp forOp,
72  ArrayRef<Value> numWorkGroups,
73  ArrayRef<Value> workGroupSizes);
