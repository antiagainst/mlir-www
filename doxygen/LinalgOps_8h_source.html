29 /// Returns the name mangled library call name to disambiguate between different
30 /// overloads at the C level. The name mangling scheme is basic and uses MLIR
31 /// type names:
32 /// 1. form a string which is the concatenation of the linalg op name with all
33 /// the operand type names, separate by underscores;
34 /// 2. drop the `linalg.` prefix, and the `<`, `>`, `?` symbols from the type.
35 /// Assumes `op` is a LinalgOp.
36 ///
37 /// Examples:
38 ///
39 /// 1. linalg.fill(%A, %f) : memref<f32>, f32
40 /// name mangles into `linalg_fill_viewf32_f32_impl`
41 ///
42 /// 2. linalg.dot(%A, %B, %C) :
43 /// memref<?xf32, stride_specification>,
44 /// memref<?xf32, stride_specification>, memref<f32>
45 /// name mangles into `linalg_dot_viewxf32_viewxf32_viewf32_impl`
46 ///
47 /// 3. linalg.matmul(...) :
48 /// memref<?x?xf32, stride_specification>,
49 /// memref<?x?xf32, stride_specification>,
50 /// memref<?x?xf32, stride_specification>
51 /// name mangles into `linalg_matmul_viewxxf32_viewxxf32_viewxxf32_impl`
52 std::string generateLibraryCallName(Operation *op);

54 /// Returns the list of maps that map loops to operands of a Linalg op.
55 /// The i-th affine map identifies loop indices to subscripts that are used when
56 /// accessing the i-th operand.
57 /// For instance, a matmul that can be written in index notation as:
58 /// `A(i, k) * B(k, j) -> C(i, j)` will have the following, ordered, list of
59 /// affine maps:
60 ///
61 /// ```mlir
62 /// (
63 /// (i, j, k) -> (i, k),
64 /// (i, j, k) -> (k, j),
65 /// (i, j, k) -> (i, j)
66 /// )
67 /// ```
68 ///
69 /// Only permutation maps are currently supported.
70 SmallVector<AffineMap, 4> loopToOperandRangesMaps(Operation *op);
